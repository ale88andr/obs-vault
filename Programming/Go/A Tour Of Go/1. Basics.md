## 1.1 Пакеты

Пакеты Каждая программа на `Go` состоит из пакетов.

Программы начинают выполнение с пакета `main`.

```go
package main

import (
	"fmt"
	"math/rand"
)

func main() {
	fmt.Println("My favorite number is", rand.Intn(10))
}
```

Эта программа использует пакеты с путями импорта `"fmt"` и `"math/rand"`.

По соглашению имя пакета совпадает с последним элементом пути импорта. Например, пакет `"math/rand"` включает файлы, начинающиеся с инструкции `package rand`.

## 1.2 Импорты 

Этот код группирует импорты в скобочную, «факторизованную» инструкцию import.

```go
package main

import (
	"fmt"
	"math"
)

func main() {
	fmt.Printf("Now you have %g problems.\n", math.Sqrt(7))
}
```

Также можно писать несколько инструкций импорта, например:

```go
import "fmt"
import "math"
```

Но хорошим стилем считается использование факторизованной инструкции `import`.

## 1.3 Экспортируемые имена 

В Go имя считается экспортируемым, если оно начинается с заглавной буквы. Например, `Pizza` — это экспортируемое имя, как и `Pi`, которое экспортируется из пакета `math`.

```go
package main

import (
	"fmt"
	"math"
)

func main() {
	fmt.Println(math.Pi)
}
```

Имена `pizza` и `pi` не начинаются с заглавной буквы, поэтому они не экспортируются.

При импорте пакета вы можете обращаться только к его экспортируемым именам. Любые «неэкспортируемые» имена недоступны извне пакета.

## 1.4 Функции 

Функция может принимать ноль или более аргументов.

В этом примере функция `add` принимает два параметра типа `int`.

Обратите внимание, что тип указывается после имени переменной.

```go
package main

import "fmt"

func add(x int, y int) int {
	return x + y
}

func main() {
	fmt.Println(add(42, 13))
}
```

Когда два или более последовательных именованных параметра функции имеют одинаковый тип, можно опустить тип у всех, кроме последнего.

В этом примере мы сократили

```go
x int, y int
```

до

```go
x, y int
```

```go
package main

import "fmt"

func add(x, y int) int {
	return x + y
}

func main() {
	fmt.Println(add(42, 13))
}
```

## 1.5 Множественные результаты 

Функция может возвращать любое количество результатов.

Функция `swap` возвращает две строки.

```go
package main

import "fmt"

func swap(x, y string) (string, string) {
	return y, x
}

func main() {
	a, b := swap("hello", "world")
	fmt.Println(a, b)
}
```

## 1.6 Именованные возвращаемые значения 

В Go возвращаемые значения могут быть именованными. В этом случае они рассматриваются как переменные, определённые в начале функции.

Такие имена следует использовать для пояснения смысла возвращаемых значений.

Оператор `return` без аргументов возвращает именованные значения. Это называется «**голым**» (naked) возвратом.

Голые return-операторы рекомендуется использовать только в коротких функциях, как показано в приведённом примере. В длинных функциях они могут ухудшить читаемость.

```go
package main

import "fmt"

func split(sum int) (x, y int) {
	x = sum * 4 / 9
	y = sum - x
	return
}

func main() {
	fmt.Println(split(17))
}
```

## 1.7 Переменные 

Инструкция `var` объявляет список переменных; как и в списках аргументов функции, тип указывается последним.

Инструкция `var` может находиться как на уровне пакета, так и внутри функции. В этом примере показаны оба варианта.

```go
package main

import "fmt"

var c, python, java bool

func main() {
	var i int
	fmt.Println(i, c, python, java)
}
```

**Переменные с инициализаторами** 

Объявление переменной с помощью `var` может включать инициализаторы — по одному на каждую переменную.

Если инициализатор указан, тип можно опустить; переменная примет тип инициализатора.

```go
package main

import "fmt"

var i, j int = 1, 2

func main() {
	var c, python, java = true, false, "no!"
	fmt.Println(i, j, c, python, java)
}
```

**Краткие объявления переменных** 

Внутри функции можно использовать короткую форму присваивания `:=` вместо объявления переменной через `var` с неявным указанием типа.

Вне функции каждая инструкция должна начинаться с ключевого слова (`var`, `func` и т. д.), поэтому конструкция `:=` недоступна.

```go
package main

import "fmt"

func main() {
	var i, j int = 1, 2
	k := 3
	c, python, java := true, false, "no!"

	fmt.Println(i, j, k, c, python, java)
}
```

## 1.7 Базовые типы 

Базовые типы в Go:

- `bool`
- `string`
- Целые числа: `int`, `int8`, `int16`, `int32`, `int64`
- Беззнаковые целые: `uint`, `uint8`, `uint16`, `uint32`, `uint64`, `uintptr`
- `byte` — псевдоним для `uint8`
- `rune` — псевдоним для `int32`, представляет символ Unicode
- Числа с плавающей точкой: `float32`, `float64`
- Комплексные числа: `complex64`, `complex128`

Пример показывает переменные различных типов, а также то, что объявления переменных могут быть сгруппированы в блоки, как и инструкции импорта.

Типы `int`, `uint` и `uintptr` обычно имеют размер 32 бита на 32-битных системах и 64 бита на 64-битных. Если вам нужен целочисленный тип, используйте `int`, если только нет конкретной причины выбрать тип с фиксированной шириной или без знака.

```go
package main

import (
	"fmt"
	"math/cmplx"
)

var (
	ToBe   bool       = false
	MaxInt uint64     = 1<<64 - 1
	z      complex128 = cmplx.Sqrt(-5 + 12i)
)

func main() {
	fmt.Printf("Type: %T Value: %v\n", ToBe, ToBe)
	fmt.Printf("Type: %T Value: %v\n", MaxInt, MaxInt)
	fmt.Printf("Type: %T Value: %v\n", z, z)
}
```

```
Type: bool Value: false
Type: uint64 Value: 18446744073709551615
Type: complex128 Value: (2+3i)
```

**Нулевые значения** 

Переменные, объявленные без явного начального значения, получают нулевое значение.

Нулевое значение:

- `0` — для числовых типов
- `false` — для логического типа
- `""` (пустая строка) — для строковых типов

```go
package main

import "fmt"

func main() {
	var i int
	var f float64
	var b bool
	var s string
	fmt.Printf("%v %v %v %q\n", i, f, b, s)
}
```

```
0 0 false ""
```

## 1.8 Преобразование типов 

Выражение `T(v)` преобразует значение `v` к типу `T`.

Примеры числовых преобразований:

```go
var i int = 42
var f float64 = float64(i)
var u uint = uint(f)
```

Или, проще:

```go
i := 42
f := float64(i)
u := uint(f)
```

В отличие от языка `C`, в `Go` присваивание между переменными разных типов требует явного преобразования. Попробуйте убрать преобразования `float64` или `uint` в примере и посмотрите, что произойдёт.

```go
package main

import (
	"fmt"
	"math"
)

func main() {
	var x, y int = 3, 4
	var f float64 = math.Sqrt(float64(x*x + y*y))
	var z uint = uint(f)
	fmt.Println(x, y, z)
}
```

**Вывод типа** 

При объявлении переменной без явного указания типа (с помощью синтаксиса `:=` или `var = выражение`), тип переменной выводится из значения **справа**.

Если правая часть объявления имеет определённый тип, новая переменная получает тот же тип:

```go
var i int
j := i // j имеет тип int
```

Но если правая часть содержит неявную числовую константу, новая переменная может получить тип `int`, `float64` или `complex128` — в зависимости от точности константы:

```go
i := 42           // int
f := 3.142        // float64
g := 0.867 + 0.5i // complex128
```

Попробуйте изменить начальное значение переменной `v` в примере и посмотрите, как это влияет на её тип.

```go
package main

import "fmt"

func main() {
	v := "42" // change me!
	fmt.Printf("v is of type %T\n", v)
}
```

## 1.9 Константы 

Константы объявляются так же, как переменные, но с использованием ключевого слова `const`.

Константы могут быть символьными, строковыми, логическими или числовыми значениями.

> Константы нельзя объявлять с помощью синтаксиса `:=`.

```go
package main

import "fmt"

const Pi = 3.14

func main() {
	const World = "世界"
	fmt.Println("Hello", World)
	fmt.Println("Happy", Pi, "Day")

	const Truth = true
	fmt.Println("Go rules?", Truth)
}
```

**Числовые константы** 

Числовые константы в Go обладают высокой точностью.

Неявно типизированная константа принимает тот тип, который требуется в текущем контексте.

(Тип `int` может хранить максимум 64-битное целое число, а иногда и меньше — в зависимости от архитектуры системы.)

```go
package main

import "fmt"

const (
	// Create a huge number by shifting a 1 bit left 100 places.
	// In other words, the binary number that is 1 followed by 100 zeroes.
	Big = 1 << 100
	// Shift it right again 99 places, so we end up with 1<<1, or 2.
	Small = Big >> 99
)

func needInt(x int) int { return x*10 + 1 }

func needFloat(x float64) float64 {
	return x * 0.1
}

func main() {
	fmt.Println(needInt(Small))
	fmt.Println(needFloat(Small))
	fmt.Println(needFloat(Big))
}
```

# 2. Операторы управления потоком: `for`, `if`, `else`, `switch` и `defer`

## 2.1 Цикл `for` 

В Go существует только одна конструкция для организации циклов — это цикл `for`.

Базовая форма цикла `for` состоит из трёх компонентов, разделённых точками с запятой:

- **инициализация (init statement)** — выполняется перед первой итерацией
- **условие (condition expression)** — проверяется перед каждой итерацией
- **пост-действие (post statement)** — выполняется после каждой итерации

Пример:

```go
package main

import "fmt"

func main() {
	sum := 0
	for i := 0; i < 10; i++ {
		sum += i
	}
	fmt.Println(sum)
}
```

Инициализация часто представляет собой краткое объявление переменной, и такие переменные доступны только внутри тела цикла `for`.

Цикл завершится, когда логическое условие станет ложным.

🔎 **Важно**: В отличие от языков C, Java или JavaScript, в Go нет круглых скобок вокруг трёх компонентов цикла `for`, но фигурные скобки `{}` обязательны.

**Инициализирующее и пост-выражение являются необязательными.**

```go
package main

import "fmt"

func main() {
	sum := 1
	for ; sum < 1000; {
		sum += sum
	}
	fmt.Println(sum)
}
```

`for` — это Go-эквивалент `while` На этом этапе можно опустить точки с запятой: `while` из языка C в Go записывается как `for`.

```go
package main

import "fmt"

func main() {
	sum := 1
	for sum < 1000 {
		sum += sum
	}
	fmt.Println(sum)
}
```

**Бесконечный цикл** 

Если опустить условие цикла, он будет выполняться бесконечно.

```go
package main

func main() {
	for {
	}
}
```

## 2.2 `if` 

Операторы `if` в Go похожи на циклы `for`: выражение не обязательно заключать в круглые скобки `( )`, но фигурные скобки `{ }` обязательны.

```go
package main

import (
	"fmt"
	"math"
)

func sqrt(x float64) string {
	if x < 0 {
		return sqrt(-x) + "i"
	}
	return fmt.Sprint(math.Sqrt(x))
}

func main() {
	fmt.Println(sqrt(2), sqrt(-4))
}
```

**`if` с коротким выражением**

Как и `for`, оператор `if` может начинаться с короткого выражения, которое выполняется перед проверкой условия.

Переменные, объявленные в этом операторе, доступны только в пределах самого `if` — до его завершения.

```go
package main

import (
	"fmt"
	"math"
)

func pow(x, n, lim float64) float64 {
	if v := math.Pow(x, n); v < lim {
		return v
	}
	return lim
}

func main() {
	fmt.Println(
		pow(3, 2, 10),
		pow(3, 3, 20),
	)
}
```

*(Попробуйте использовать переменную `v` в последнем операторе `return` — она будет недоступна вне блока `if`.)

**`if` и `else`** 

Переменные, объявленные внутри короткого оператора `if`, также доступны внутри любого блока `else`.

(Оба вызова функции `pow` возвращают свои результаты до того, как начнётся вызов `fmt.Println` в `main`.)

```go
package main

import (
	"fmt"
	"math"
)

func pow(x, n, lim float64) float64 {
	if v := math.Pow(x, n); v < lim {
		return v
	} else {
		fmt.Printf("%g >= %g\n", v, lim)
	}
	// can't use v here, though
	return lim
}

func main() {
	fmt.Println(
		pow(3, 2, 10),
		pow(3, 3, 20),
	)
}
```

## 2.3 `switch` 

Оператор `switch` — это более короткий способ записать последовательность операторов `if - else`. Он выполняет первый `case`, значение которого равно выражению условия.

Оператор `switch` в `Go` похож на оператор switch в` C, C++, Java, JavaScript` и `PHP`, за исключением того, что `Go` выполняет только выбранный `case`, а не все последующие. По сути, оператор `break`, который требуется в конце каждого `case` в этих языках, в `Go` предоставляется автоматически. Другое важное отличие состоит в том, что выражения `case` в `Go` не обязательно должны быть константами, а значения не обязательно должны быть целыми числами.

```go
package main

import (
	"fmt"
	"runtime"
)

func main() {
	fmt.Print("Go runs on ")
	switch os := runtime.GOOS; os {
	case "darwin":
		fmt.Println("macOS.")
	case "linux":
		fmt.Println("Linux.")
	default:
		// freebsd, openbsd,
		// plan9, windows...
		fmt.Printf("%s.\n", os)
	}
}
```

**Порядок вычисления в `switch`** 

Оператор `switch` проверяет варианты (`case`) сверху вниз и останавливается, когда находит подходящий.

(Например:

```go
switch i {
case 0:
case f():
}
```

не вызывает `f`, если `i == 0`.)

```go
package main

import (
	"fmt"
	"time"
)

func main() {
	fmt.Println("When's Saturday?")
	today := time.Now().Weekday()
	switch time.Saturday {
	case today + 0:
		fmt.Println("Today.")
	case today + 1:
		fmt.Println("Tomorrow.")
	case today + 2:
		fmt.Println("In two days.")
	default:
		fmt.Println("Too far away.")
	}
}
```

**`switch` без условия** 

`switch` без условия — это то же самое, что `switch true`.

Эта конструкция может быть чистым способом записи длинных цепочек `if-then-else`.

```go
package main

import (
	"fmt"
	"time"
)

func main() {
	t := time.Now()
	switch {
	case t.Hour() < 12:
		fmt.Println("Good morning!")
	case t.Hour() < 17:
		fmt.Println("Good afternoon.")
	default:
		fmt.Println("Good evening.")
	}
}
```

## 2.3 `defer` 

Оператор `defer` откладывает выполнение функции до тех пор, пока не завершится окружающая функция.

Аргументы отложенного вызова вычисляются немедленно, но сам вызов функции выполняется только после возврата из окружающей функции.

```go
package main

import "fmt"

func main() {
	defer fmt.Println("world")

	fmt.Println("hello")
}
```

**Многочисленные `defer`** 

Отложенные вызовы функций помещаются в стек. Когда функция завершает выполнение, её отложенные вызовы выполняются в порядке **последним пришёл — первым выполнен** (LIFO).

```go
package main

import "fmt"

func main() {
	fmt.Println("counting")

	for i := 0; i < 10; i++ {
		defer fmt.Println(i)
	}

	fmt.Println("done")
}
```

```
counting
done
9
8
7
6
5
4
3
2
1
0
```

# 3. Cтруктуры, срезы и отображения.

## 3.1 Указатели

Указатели В Go есть указатели. Указатель хранит адрес в памяти, по которому находится значение.

Тип `*T` — это указатель на значение типа `T`. Его нулевое значение — `nil`.

```go
var p *int
```

Оператор `&` создаёт указатель на свой операнд:

```go
i := 42
p = &i
```

Оператор `*` обозначает значение, на которое указывает указатель:

```go
fmt.Println(*p) // чтение i через указатель p
*p = 21         // установка i через указатель p
```

Это называется **разыменованием** или **косвенным доступом**.

В отличие от C, в Go **нет арифметики указателей**.

```go
package main

import "fmt"

func main() {
	i, j := 42, 2701

	p := &i         // point to i
	fmt.Println(*p) // read i through the pointer
	*p = 21         // set i through the pointer
	fmt.Println(i)  // see the new value of i

	p = &j         // point to j
	*p = *p / 37   // divide j through the pointer
	fmt.Println(j) // see the new value of j
}
```

```
42
21
73
```

## 3.2 Структуры 

Структура — это набор полей.

```go
package main

import "fmt"

type Vertex struct {
	X int
	Y int
}

func main() {
	fmt.Println(Vertex{1, 2}) // {1 2}
}
```

**Поля структуры** 

Поля структуры доступны через точку.

```go
package main

import "fmt"

type Vertex struct {
	X int
	Y int
}

func main() {
	v := Vertex{1, 2}
	v.X = 4
	fmt.Println(v.X)
}
```

**Указатели на структуры** 

Поля структуры можно получить через указатель на структуру.

Чтобы обратиться к полю `X` структуры, имея указатель `p`, можно написать `(*p).X`. Однако такая запись громоздка, поэтому язык позволяет использовать просто `p.X` — без явного разыменования.

```go
package main

import "fmt"

type Vertex struct {
	X int
	Y int
}

func main() {
	v := Vertex{1, 2}
	p := &v
	p.X = 1e9 // 1000000000
	fmt.Println(v) // {1000000000 2}
}
```

**Литералы структур** 

Литерал структуры обозначает заново выделенное значение структуры путём перечисления значений её полей.

Можно указать только часть полей, используя синтаксис `Name:`. (Порядок именованных полей не имеет значения.)

Специальный префикс `&` возвращает указатель на значение структуры.

```go
package main

import "fmt"

type Vertex struct {
	X, Y int
}

var (
	v1 = Vertex{1, 2}  // has type Vertex
	v2 = Vertex{X: 1}  // Y:0 is implicit
	v3 = Vertex{}      // X:0 and Y:0
	p  = &Vertex{1, 2} // has type *Vertex
)

func main() {
	fmt.Println(v1, p, v2, v3) // {1 2} &{1 2} {1 0} {0 0}
}
```

## 3.3 Массивы 

Тип `[n]T` — это массив из `n` значений типа `T`.

Выражение:

```go
var a [10]int
```

объявляет переменную `a` как массив из десяти целых чисел.

Длина массива является частью его типа, поэтому массивы **нельзя изменять по размеру**. Это может показаться ограничением, но не волнуйтесь — Go предоставляет удобный способ работы с массивами.

```go
package main

import "fmt"

func main() {
	var a [2]string
	a[0] = "Hello"
	a[1] = "World"
	fmt.Println(a[0], a[1])
	fmt.Println(a)

	primes := [6]int{2, 3, 5, 7, 11, 13}
	fmt.Println(primes)
}
```

## 3.4 Срезы 

Массив имеет фиксированный размер. Срез, с другой стороны, — это гибкое представление элементов массива с динамическим размером. На практике срезы гораздо чаще используются, чем массивы.

Тип `[]T` — это срез с элементами типа `T`.

Срез формируется путём указания двух индексов — нижней и верхней границы — через двоеточие:

```go
a[low : high]
```

Это выбирает полуоткрытый диапазон, который включает первый элемент, но исключает последний.

Следующее выражение создаёт срез, включающий элементы с `1` по `3` массива `a`:

```go
a[1:4]
```

```go
package main

import "fmt"

func main() {
	primes := [6]int{2, 3, 5, 7, 11, 13}

	var s []int = primes[1:4]
	fmt.Println(s)
}
```

**Срезы** похожи на ссылки на массивы Срез не хранит данные — он просто описывает участок базового массива.

Изменение элементов среза изменяет соответствующие элементы базового массива.

Другие срезы, использующие тот же базовый массив, увидят эти изменения.

```go
package main

import "fmt"

func main() {
	names := [4]string{
		"John",
		"Paul",
		"George",
		"Ringo",
	}
	fmt.Println(names) // [John Paul George Ringo]

	a := names[0:2]
	b := names[1:3]
	fmt.Println(a, b) // [John Paul] [Paul George]

	b[0] = "XXX"
	fmt.Println(a, b) // [John XXX] [XXX George]
	fmt.Println(names) // [John XXX George Ringo]
}
```

**Литералы срезов** 

Литерал среза похож на литерал массива, но без указания длины.

Вот литерал массива:

```go
[3]bool{true, true, false}
```

А вот это создаёт тот же массив, а затем формирует срез, который на него ссылается:

```go
[]bool{true, true, false}
```

```go
package main

import "fmt"

func main() {
	q := []int{2, 3, 5, 7, 11, 13}
	fmt.Println(q)

	r := []bool{true, false, true, true, false, true}
	fmt.Println(r)

	s := []struct {
		i int
		b bool
	}{
		{2, true},
		{3, false},
		{5, true},
		{7, true},
		{11, false},
		{13, true},
	}
	fmt.Println(s)
}
```

Значения по умолчанию при срезах

При создании среза можно опустить верхнюю или нижнюю границу, чтобы использовать значения по умолчанию.

- Значение по умолчанию для нижней границы — `0`.
- Значение по умолчанию для верхней границы — длина среза.

Для массива:

```go
var a [10]int
```

все следующие выражения создают один и тот же срез:

```go
a[0:10]
a[:10]
a[0:]
a[:]
```

```go
package main

import "fmt"

func main() {
	s := []int{2, 3, 5, 7, 11, 13}

	s = s[1:4]
	fmt.Println(s) // [3 5 7]

	s = s[:2]
	fmt.Println(s) // [3 5]

	s = s[1:]
	fmt.Println(s) // [5]
}
```

**Длина и вместимость среза** 

Срез имеет как **длину**, так и **вместимость**.

- **Длина** среза — это количество элементов, которые он содержит.
- **Вместимость** — это количество элементов в базовом массиве, начиная с первого элемента среза.

Длину и вместимость среза `s` можно получить с помощью выражений `len(s)` и `cap(s)`.

Можно увеличить длину среза, пересрезав его — если позволяет вместимость. Попробуйте изменить одну из операций со срезом в примере программы, чтобы расширить его за пределы вместимости, и посмотрите, что произойдёт.

```go
package main

import "fmt"

func main() {
	s := []int{2, 3, 5, 7, 11, 13}
	printSlice(s)

	// Slice the slice to give it zero length.
	s = s[:0]
	printSlice(s)

	// Extend its length.
	s = s[:4]
	printSlice(s)

	// Drop its first two values.
	s = s[2:]
	printSlice(s)
}

func printSlice(s []int) {
	fmt.Printf("len=%d cap=%d %v\n", len(s), cap(s), s)
}
```

```
len=6 cap=6 [2 3 5 7 11 13]
len=0 cap=6 []
len=4 cap=6 [2 3 5 7]
len=2 cap=4 [5 7]
```

Нулевой срез (`nil slice`) 

Нулевое значение среза — это `nil`.

Срез `nil` имеет **длину 0**, **вместимость 0** и **не содержит базового массива**.

```go
package main

import "fmt"

func main() {
	var s []int
	fmt.Println(s, len(s), cap(s)) // [] 0 0
	if s == nil {
		fmt.Println("nil!") // nil!
	}
}
```

**Создание среза с помощью `make`** 

Срезы можно создавать с помощью встроенной функции `make` — это способ создавать массивы с динамическим размером.

Функция `make` выделяет обнулённый массив и возвращает срез, который на него ссылается:

```go
a := make([]int, 5)  // len(a)=5
```

Чтобы задать вместимость, передайте третий аргумент:

```go
b := make([]int, 0, 5) // len(b)=0, cap(b)=5

b = b[:cap(b)] // len(b)=5, cap(b)=5
b = b[1:]      // len(b)=4, cap(b)=4
```

```go
package main

import "fmt"

func main() {
	a := make([]int, 5)
	printSlice("a", a)

	b := make([]int, 0, 5)
	printSlice("b", b)

	c := b[:2]
	printSlice("c", c)

	d := c[2:5]
	printSlice("d", d)
}

func printSlice(s string, x []int) {
	fmt.Printf("%s len=%d cap=%d %v\n",
		s, len(x), cap(x), x)
}
```

```
a len=5 cap=5 [0 0 0 0 0]
b len=0 cap=5 []
c len=2 cap=5 [0 0]
d len=3 cap=3 [0 0 0]
```

**Срезы срезов** 

Срезы могут содержать значения любого типа — включая другие срезы.

```go
package main

import (
	"fmt"
	"strings"
)

func main() {
	// Create a tic-tac-toe board.
	board := [][]string{
		[]string{"_", "_", "_"},
		[]string{"_", "_", "_"},
		[]string{"_", "_", "_"},
	}

	// The players take turns.
	board[0][0] = "X"
	board[2][2] = "O"
	board[1][2] = "X"
	board[1][0] = "O"
	board[0][2] = "X"

	for i := 0; i < len(board); i++ {
		fmt.Printf("%s\n", strings.Join(board[i], " "))
	}
}
```

```
X _ X
O _ X
_ _ O
```

**Добавление к срезу** 

Обычно к срезу добавляют новые элементы, поэтому Go предоставляет встроенную функцию `append`. Документация встроенного пакета описывает `append`.

```go
func append(s []T, vs ...T) []T
```

Первый параметр `s` функции `append` — это срез типа `T`, а остальные — значения типа `T`, которые нужно добавить к срезу.

Результатом `append` будет срез, содержащий все элементы исходного среза плюс предоставленные значения.

Если базовый массив `s` слишком мал, чтобы вместить все заданные значения, будет выделен больший массив. Возвращённый срез будет указывать на заново выделенный массив.

```go
package main

import "fmt"

func main() {
	var s []int
	printSlice(s)

	// append works on nil slices.
	s = append(s, 0)
	printSlice(s)

	// The slice grows as needed.
	s = append(s, 1)
	printSlice(s)

	// We can add more than one element at a time.
	s = append(s, 2, 3, 4)
	printSlice(s)
}

func printSlice(s []int) {
	fmt.Printf("len=%d cap=%d %v\n", len(s), cap(s), s)
}
```

```
len=0 cap=0 []
len=1 cap=1 [0]
len=2 cap=2 [0 1]
len=5 cap=6 [0 1 2 3 4]
```

## 3.5 Диапазон (`range`) 

Форма `range` цикла `for` выполняет итерацию по срезу или отображению (`map`).

При переборе среза с помощью `range` на каждой итерации возвращаются два значения. Первое — это индекс, а второе — копия элемента по этому индексу.

```go
package main

import "fmt"

var pow = []int{1, 2, 4, 8, 16, 32, 64, 128}

func main() {
	for i, v := range pow {
		fmt.Printf("2**%d = %d\n", i, v)
	}
}
```

Можно пропустить индекс или значение, присвоив их переменной `_`.

```go
for i, _ := range pow
for _, value := range pow
```

Если нужен только индекс, можно опустить вторую переменную:

```go
for i := range pow
```

```go
package main

import "fmt"

func main() {
	pow := make([]int, 10)
	for i := range pow {
		pow[i] = 1 << uint(i) // == 2**i
	}
	for _, value := range pow {
		fmt.Printf("%d\n", value)
	}
}
```

Пример:

```go
package main

import "golang.org/x/tour/pic"

func Pic(dx, dy int) [][]uint8 {
	result := make([][]uint8, dy)
	for y:=0; y<dy; y++ {
		row := make([]uint8, dx)
		for x:=0; x<dx; x++ {
			row[x] = uint8((x+y)/2)
		}
		result[y] = row
	}
	return result
}

func main() {
	pic.Show(Pic)
}
```

или с использованием `range`:

```go
package main

import "golang.org/x/tour/pic"

func Pic(dx, dy int) [][]uint8 {
	result := make([][]uint8, dy)
	for y := range(dy) {
		row := make([]uint8, dx)
		for x := range(dx) {
			row[x] = uint8((x^y))
		}
		result[y] = row
	}
	return result
}

func main() {
	pic.Show(Pic)
}
```

## 3.6 Отображения (`Maps`) 

Отображение сопоставляет ключи со значениями.

Нулевое значение отображения — `nil`. Отображение `nil` не содержит ключей, и в него нельзя добавлять ключи.

Функция `make` возвращает отображение заданного типа, инициализированное и готовое к использованию.

```go
package main

import "fmt"

type Vertex struct {
	Lat, Long float64
}

var m map[string]Vertex

func main() {
	m = make(map[string]Vertex)
	m["Bell Labs"] = Vertex{
		40.68433, -74.39967,
	}
	fmt.Println(m["Bell Labs"]) // {40.68433 -74.39967}
}
```

**Литералы отображений** 

Литералы отображений похожи на литералы структур, но ключи обязательны.

```go
package main

import "fmt"

type Vertex struct {
	Lat, Long float64
}

var m = map[string]Vertex{
	"Bell Labs": Vertex{
		40.68433, -74.39967,
	},
	"Google": Vertex{
		37.42202, -122.08408,
	},
}

func main() {
	fmt.Println(m)
}
```

Если тип верхнего уровня — это просто имя типа, его можно опустить в элементах литерала.

```go
package main

import "fmt"

type Vertex struct {
	Lat, Long float64
}

var m = map[string]Vertex{
	"Bell Labs": {40.68433, -74.39967},
	"Google":    {37.42202, -122.08408},
}

func main() {
	fmt.Println(m)
}
```

**Изменение отображений (Mutating Maps)** 

Вставить или обновить элемент в отображении `m`:

```go
m[key] = elem
```

Получить элемент:

```go
elem = m[key]
```

Удалить элемент:

```go
delete(m, key)
```

Проверить наличие ключа с помощью присваивания двух значений:

```go
elem, ok = m[key]
```

Если ключ есть в `m`, то `ok` будет `true`. Если нет — `false`.

Если ключ отсутствует в отображении, то `elem` будет нулевым значением для типа элементов отображения.

Примечание: Если `elem` или `ok` ещё не объявлены, можно использовать короткую форму объявления:

```go
elem, ok := m[key]
```

```go
package main

import "fmt"

func main() {
	m := make(map[string]int)

	m["Answer"] = 42
	fmt.Println("The value:", m["Answer"])

	m["Answer"] = 48
	fmt.Println("The value:", m["Answer"])

	delete(m, "Answer")
	fmt.Println("The value:", m["Answer"])

	v, ok := m["Answer"]
	fmt.Println("The value:", v, "Present?", ok)
}
```
