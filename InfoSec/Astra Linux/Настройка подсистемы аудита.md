### Архитектура аудита

- Регистрацией событий занимается служба `auditd`
- Поддерживается:
	•	файловый аудит — протоколирование для объектов файловой системы
	•	аудит процессов — протоколирование для пользователей
- Дополнительный модуль `pam_parsec_aud`

Модуль `PARSEC` перехватывает и передает события в `auditd`

### Правила протоколирования

- Два типа правил:
	- постоянные — действуют всегда, в том числе после перезагрузки ОС, задаются в файлах формата `*.rules`, в каталоге `/etc/аudit/rules.d/`
	- временные — действуют до перезагрузки ОС, задаются с помощью утилиты `auditctl`
- Правила, необходимые для работы встроенного аудита, заданы в файле `/etc/audit/rules.d/10-ра rsec.rules`
- Описание правил: `man auditctl`

```shell
# ls /etc/audit
auditd.conf # Основной файл конфигурации службы
audit.rules 
audit.rules.prev 
audit-stop.rules 
plugins.d/ 
rules.d/
```
