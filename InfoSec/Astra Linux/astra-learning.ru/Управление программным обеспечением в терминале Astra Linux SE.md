Программное обеспечение для ОС Linux распространяется в программных пакетах. Пакет представляет собой архивный файл, содержащий программный код в бинарном или исходном виде, а также метаданные о программе, её версии, зависимостях и другую информацию.

Имя пакету присваивается в соответствии с соглашением и обычно имеет следующий вид:

﻿ ```
﻿ <имя_пакета>-<версия>-<архитектура>.deb
﻿ ```

Каждый программный пакет содержит только необходимый минимум файлов и библиотек, все остальные необходимые ему компоненты указываются в нём в качестве зависимостей. Зависимость — потребность программного пакета в ресурсах, которые находятся в составе других пакетов.

Связи с другими пакетами хранятся в метаданных. Вот основные поля.
- **Зависит** (_Depends_).Обязательные пакеты. Без них программа не будет работать.
- **Рекомендует** (_Recommends_).Необязательные, но полезные пакеты. Они значительно расширяют возможности программы.
- **Предлагает** (_Suggests_).Пакеты для дополнительной функциональности. Устанавливать их или нет — на ваше усмотрение.
- **Конфликтует** (_Conflicts_).Пакеты, которые нельзя устанавливать вместе. Они мешают работе друг друга.

## Репозитории

В _Astra Linux_ все программы и обновления распространяются в виде **deb-пакетов** из специальных хранилищ — **репозиториев**.

**Репозиторий** — это централизованное сетевое хранилище, которое служит для установки нового и обновления уже установленного ПО.

##### Структура репозитория

- **pool** — каталог, где хранятся сами _deb-файлы_ (программы).
- **dists** — каталог с метаданными (описание пакетов и репозитория).

##### Компоненты репозитория (по лицензии)

- **main** — свободное ПО.
- **contrib** — свободное ПО, но для работы требует проприетарное.
- **non-free** — проприетарное (несвободное) ПО.
- **non-free-firmware** — прошивки для оборудования.

## Инструменты для работы с пакетами в Astra Linux

В Astra Linux для управления ПО используются разные инструменты.

##### Для deb-пакетов (из репозиториев Astra)
- **apt** — основной и самый современный инструмент. Работает с репозиториями, устанавливает программы, обновляет систему и автоматически разрешает зависимости.
- **dpkg** — утилита для прямой работы с deb-файлами. Устанавливает, удаляет и распаковывает пакеты, но **не умеет** работать с репозиториями и разрешать зависимости. Используется для установки программ, скачанных вручную.
- **aptitude** — устаревший менеджер с псевдографическим интерфейсом (меню). По функциям похож на apt, но сейчас используется редко.

##### Универсальные форматы (Snap, Flatpak, AppImage)
Эти форматы позволяют запускать одно и то же приложение на разных дистрибутивах Linux.
- **Snap** — пакеты от Ubuntu. Запускаются в «песочнице» (изолированно) и устанавливаются через магазин Snap Store.
- **Flatpak** — аналог Snap, но с полностью открытым исходным кодом. Также работает в изолированном окружении.
- **AppImage** — портативный формат. Приложение просто запускается из файла, установка в систему не требуется.

## Просмотр информации о репозиториях

Чтобы указать программам-установщикам репозитории, из которых следует загружать устанавливаемые программные пакеты, необходимо:
- прописать эти репозитории в файл `sources.list` (путь к файлу: `/etc/apt/sources.list`);
- или создать файлы с произвольными именами и суффиксом `.list`, расположить их в каталоге `/etc/apt/sources.list.d` и прописать в них нужные репозитории.

При этом для каждого репозитория в файле необходимо сформировать строку вида:

﻿ ```
﻿ deb <путь_к_корневому каталогу_репозитория> <код_дистрибутива> <список_компонент>
﻿ ```

Каждая такая запись содержит:
- тип архива — *deb* (бинарные пакеты) или *deb-src* (исходные коды);;
- путь к репозиторию — локальный или сетевой;
- код дистрибутива (название репозитория);
- используемые компоненты.

Просмотреть список прописанных в файле sources.list репозиториев, можно при помощи команды:  `cat /etc/apt/sources.list`

> Обозначения репозиториев:
> - путь к репозиторию, расположенному на оптическом носителе, начинается с `cdrom`; 
> - путь к сетевому репозиторию начинается с `https` или `ftp`;
> - путь к локальным репозиториям начинается с `file`;
> - неиспользуемые (неактивные) репозитории закомментированы в начале строки символом `#`.

По умолчанию (из соображений безопасности), после установки ОС сетевые репозитории для установки и обновления программного обеспечения отключены. Активным является только репозиторий `cdrom`. Чтобы активировать нужный репозиторий, необходимо раскомментировать соответствующую строку в файле `sources.list` (убрать символ `#` в начале строки с описанием репозитория). Неиспользуемый репозиторий нужно закомментировать.

> После каждого внесения изменений в файлы с описанием репозиториев следует обновить списки пакетов командой  `sudo apt update` , чтобы соответствующие изменения были применены.

## Основные интернет-репозитории Astra Linux 1.8

Интернет-репозитории Astra Linux предназначены в том числе и для обеспечения процедуры обновления ОС при наличии доступа в интернет и позволяют выполнять установку и обновление пакетов с помощью стандартных установщиков пакетов — графического менеджера пакетов synaptic и инструмента командной строки apt.

Для версии Astra Linux Special Edition 1.8 предоставляются следующие репозитории:

## Установка оперативных и срочных обновлений

Для установки оперативных обновлений ОС в списке репозиториев должен быть указан основной репозиторий (main-repository), содержащий устанавливаемое обновление.

Для минимизации усилий поддержания ОС в актуальном состоянии в части оперативных и срочных обновлений достаточно использовать репозиторий:

﻿```shell
﻿deb https://dl.astralinux.ru/astra/stable/1.8_x86-64/main-repository/1.8_x86-64 main contrib non-free non-free-firmware
﻿```

При этом для установки оперативных и срочных обновлений основным используется поддерево интернет-репозиториев frozen, содержащее фиксированные ссылки, публикуемые в официальных *бюллетенях безопасности*. Установку оперативных обновлений следует выполнять по инструкциям, опубликованным в данных бюллетенях.

> *Бюллетень безопасности* — это документ, в котором публикуются оперативные обновления безопасности для устранения уязвимостей в системе. Бюллетени описывают, какие уязвимости были устранены в конкретном обновлении, и содержат методики для их устранения.

## Подключение стороннего репозитория

Для расширения списка доступного для установки программного обеспечения имеется возможность подключения сторонних интернет-репозиториев, например пакетной базы `Debian 12 bookworm`. При этом для подключения стороннего репозитория должен быть установлен публичный ключ, которым подписаны пакеты данного репозитория.

Для подключения репозитория Debian, пакет с публичными ключами которого находится в репозитории Astra Linux, необходимо:

1. Добавить ключ репозитория с помощью команды:  `sudo apt install debian-archive-keyring` ﻿
2. Добавить в файл со списком репозиториев `/etc/apt/sources.list` строку с описанием репозитория:
﻿ `deb https://deb.debian.org/debian/ bookworm main contrib non-free` ﻿
3. Выполнить команду:  `sudo apt update`

После подключения репозитория Debian пользователю станет доступен более широкий список программных пакетов для установки.

## Создание и подключение собственного репозитория

В некоторых случаях может потребоваться создание и подключение собственного репозитория. Это становится актуальным тогда, когда для работы требуется программное обеспечение, которое не включено в официальные репозитории дистрибутива или разработано самостоятельно. В этих случаях для хранения и распространения программных пакетов создаётся собственный отдельный репозиторий, в котором могут содержаться только необходимые в работе программные пакеты.

Рассмотрим пример создания собственного репозитория на FTP-сервере. Для этого необходимо выполнить следующие действия:
1. Установить программное обеспечение FTP-сервера `vsftpd`:  `sudo apt install vsftpd` 
2. Проверить статус FTP-сервера, выполнив команду:  `sudo systemctl status vsftpd`
3. В файле `/etc/vsftpd.conf` настроить работу FTP-сервера в режиме, разрешающем анонимную аутентификацию (значение параметра anonymous_enable должно быть YES). Проверить это можно с помощью команды:  `egrep -v '^(#|$)' /etc/vsftpd.conf`
![[Pasted image 20260519130913.png]]

При необходимости внести необходимую корректировку в файл `/etc/vsftpd.conf`, открыв его в редакторе `Nano`.
4. Выполнить команду, которая поможет FTP-серверу перечитать изменённый конфигурационный файл:  `sudo systemctl reload vsftp` ﻿
5. Установить утилиту reprepro (если не установлена). С её помощью осуществляется добавление программных пакетов в репозиторий:  `sudo apt install reprepro` ﻿
6. Создать новый каталог (astra/conf) в каталоге /srv/ftp, в котором будет размещаться репозиторий:  `sudo mkdir -p /srv/ftp/astra/conf` ﻿
7. Создать конфигурационный файл distributions с описанием структуры и параметров репозитория в подкаталоге conf при помощи команды:  `sudo nano /srv/ftp/astra/conf/distributions`
```
Origin: Debian
Codename: 1.8_x86-64
Suite: stable
Version: 1.8.2
Architectures: amd64
Components: main contrib non-free non-free-firmware
SignWith: yes
```
8. Создать ключ для репозитория.
	Процедура создания ключа:
	- Выполнить формирование таблицы доверия:  `sudo gpg --list-keys` ﻿
	- Выполнить команду  `sudo gpg --gen-key` ﻿
	- На запрос имени указать его (например, MyOrg).
	- На запрос адреса электронной почты, указать его, например test@astralinux.ru.
>   Сгенерированные ключи размещаются в каталоге `/root/.gnupg`.
9. Создать структуру для репозитория. Для создания собственного репозитория используется утилита `reprepro`:

﻿ ```
﻿ sudo reprepro --ask-passphrase -b /srv/ftp/astra/ export
﻿ ```
