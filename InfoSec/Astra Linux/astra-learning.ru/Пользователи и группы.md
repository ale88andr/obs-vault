
## Введение

**Учётная запись** представляет собой совокупность информации, которая позволяет системе идентифицировать пользователя (группу пользователей) и управлять его доступом к функциям, ресурсам и данным в информационных системах

Учётная запись пользователя выполняет следующие функции:
- **идентификация** — позволяет системе точно определить, какой пользователь обращается к её ресурсам;
- **персонализация** — даёт возможность сохранять настройки и предпочтения, улучшая удобство работы;
- **безопасность** — обеспечивает защиту информации и контроль доступа к конфиденциальным данным.

## Идентификаторы пользователей и групп

В ОС Astra Linux предусмотрено три типа пользователей:
- суперпользователь — имя `root`, домашний каталог `/root`. Суперпользователь имеет полный доступ к системе. В Astra Linux по умолчанию пароль для суперпользователя не задан и ему заблокирован локальный вход в систему, поэтому войти в систему с использованием учётной записи `root` нельзя;
- обычные пользователи — учётные записи пользователей, которым разрешён терминальный вход в систему. Домашние каталоги пользователей по умолчанию размещаются в `/home`;
- системные пользователи — учётные записи, которые нужны для функционирования операционной системы и служб.

Каждому пользователю присваивается свой идентификатор (`UserID`, далее — `UID`):
- суперпользователь имеет `UID=0`;
- обычные пользователи имеют идентификаторы `UID` с номерами от `1000` до `59999`;
- системные пользователи имеют учётные записи с идентификаторами UID от `0` до `99`, которые используются самой ОС и одинаковы для всех систем, а также учётные записи с `UID` от `100` до `999`, которые используются службами, создаются динамически и могут отличаться в разных системах.

Пользователей, как правило, объединяют в группы. Каждой группе пользователей присваивается свой идентификатор (`GroupID`, далее — `GID`). Идентификаторы распределяются следующим образом:
- `GID` от `0` до `99` используются самой системой,
- `GID` от `100` до `999` используются службами и создаются динамически,
- `GID` от `1000` до `59999` применяются для пользовательских групп и распределяются динамически.

Прежде чем приступить к непосредственному созданию учётных записей, рекомендуется провести ряд подготовительных мероприятий:
- выработать определённые правила формирования имён учётных записей пользователей и групп пользователей;
- решить, каким образом будут распределяться идентификаторы учётных записей пользователей и групп (UID, GID);
- определить основную (первичную) и дополнительные (вторичные) группы для учётных записей пользователей;
- решить, где будут располагаться домашние каталоги пользователей;
- решить, какой командный интерпретатор будет использоваться по умолчанию;
- определить правила формирования и политику использования паролей.

## База данных локальных учётных записей

Все данные локальных учётных записей пользователей и групп хранятся в следующих файлах:
- `/etc/passwd` — учётные записи пользователей;
- `/etc/shadow` — пароли пользователей и атрибуты паролей;
- `/etc/group` — учётные записи групп;
- `/etc/gshadow` — пароли для групп.

## Структура файла `/etc/passwd`

Информация об учётных записях пользователей хранится в файле `/etc/passwd`. Каждая строка в файле соответствует одной учётной записи.

![[Pasted image 20260513170338.png]]

Поля информации (параметры учётной записи) разделяются двоеточием (:). Каждая строка файла /etc/passwd содержит семь полей:

1. Имя учётной записи пользователя (логин, username).
2. Пароль в зашифрованном виде. Символ Х означает, что пароль хранится в отдельном файле — `/etc/shadow`.
3. Идентификатор пользователя (UID).
4. Идентификатор группы (GID), которой принадлежит пользователь. Пользователю должна быть назначена первичная группа. Дополнительные (вторичные) группы перечислены в файле /etc/group.
5. Подробная информация о пользователе (поле GECOS). Значения записываются через запятую без использования символа двоеточия. Обычно значения идут в следующем порядке:
	- полное имя пользователя,
	- адрес,
	- номер рабочего телефона,
	- номер домашнего телефона,
	- прочая информация (например, электронная почта).
6. Абсолютный путь к домашнему каталогу пользователя (home directory). По умолчанию домашние каталоги пользователей именуются в соответствии с именем учётной записи пользователей и создаются в каталоге `/home`.
7. Первая программа, выполняемая сразу после входа пользователя в систему. Обычно это командный интерпретатор (login shell), например `/bin/bash`. Если указать специальное значение `/bin/false` (которое ничего не делает и возвращает контроль системе немедленно), то пользователь не сможет войти в систему.

Файл `/etc/passwd` может быть прочитан обычным пользователем, что является уязвимым местом в безопасности. Получив этот файл, можно попытаться подобрать пароль к учётной записи. Поэтому пароли в зашифрованном виде хранятся в отдельном файле — `/etc/shadow`, доступа к которому обычный пользователь не имеет.

## Структура файла `/etc/shadow`

В файле `/etc/shadow` хранятся пароли в зашифрованном виде.

Формат строки:

![[Pasted image 20260513170632.png]]

Поля информации в файле `/etc/shadow` разделяются двоеточием (`:`).

Каждая строка этого файла содержит девять полей:

1. Имя пользователя.
2. Зашифрованный пароль.
3. Дата последнего изменения пароля в днях, отсчитываемых с 1 января 1970 года.
4. Минимальное число дней между изменениями пароля — количество дней, которое должно пройти, прежде чем можно будет изменить пароль пользователя. Значение `0` означает отсутствие данного ограничения.
5. Максимальное число дней между изменениями пароля — количество дней до необходимости смены пароля. Значение `99999` указывает на то, что ограничение не установлено.
6. Период предупреждения — количество дней до истечения срока действия пароля, в течение которых пользователя предупреждают о необходимости его изменения.
7. Период бездействия — количество дней бездействия пользователя после истечения срока действия пароля до блокировки учётной записи пользователя.
8. Срок действия учётной записи — дата, когда учётная запись была отключена
9. Не используется.

## Структура файла `/etc/group`

Информация о группах содержится в файле `/etc/group`.

Формат строки: `group_name : password : GID : user_list`

Строка файла `/etc/group` состоит из нескольких полей, каждое из которых отделено от другого двоеточием (`:`):

1. Имя группы.
2. Пароль в зашифрованном виде. Символ `Х` означает, что пароль хранится в отдельном файле — `/etc/gshadow`.
3. Числовой идентификатор группы (`GID`).
4. Список пользователей, разделённых запятой, для которых данная группа является дополнительной (вторичной).

## Структура файла `/etc/gshadow`

Как и в случае с пользователями, пароли для групп хранятся в отдельном файле — `/etc/gshadow`.

Для каждой группы используется отдельная строка. Формат строки:

`groupname : encrypted-password : group-admins : group-users`

Строка файла `/etc/gshadow` состоит из нескольких полей, каждое из которых отделено от другого двоеточием (`:`):

1. Имя группы.
2. Закодированный пароль группы.
3. Администраторы группы — члены группы, перечисленные через запятую, которые могут менять пароль или членство в группе.
4. Члены группы — члены группы, перечисленные в этом поле, являются обычными пользователями данной группы (тот же список, что и в `/etc/group`).

Пароль нужен для того, чтобы пользователь, не состоящий в группе, мог получить к ней доступ.

Пустое значение в поле пароля группы означает, что только члены группы могут использовать её для получения доступа к файлам.

## Управление учётными записями пользователей и групп

  

Для управления учётными записями пользователей и групп используются следующие утилиты (команды):
- `useradd` — создание учётной записи пользователя;
- `usermod` — изменение параметров учётной записи пользователя;
- `userdel` — удаление учётной записи пользователя;
- `groupadd` — создание учётной записи группы;
- `groupmod` — изменение параметров учётной записи группы;
- `groupdel` — удаление учётной записи группы.

Настройки по умолчанию, которые используются данными утилитами, содержатся в файле `/etc/login.defs`.

Информацию обо всех учётных записях, которые уже есть в системе, можно получить, воспользовавшись командой `lslogins`. Формат команды:

﻿ `lslogins <параметры> <имя пользователя>` ﻿

Утилита `lslogins` позволяет получить информацию об учётных записях пользователей, атрибутах паролей, сеансах пользователей:

![[Pasted image 20260513171433.png]]

Использование команды `lslogins` без параметров отобразит перечень всех имеющихся пользователей в порядке возрастания их `UID`. Для отображения обычных пользователей применяется параметр `-u`.

Наиболее часто используемые параметры команды `lslogins`:
- `-a`, `--acc-expiration` — вывести информацию об истечении срока действия паролей;
- `-c`, `--colon-separate` — вывести данные в формате, схожем с форматом файла /etc/passwd;
- `-G`, `--supp-groups` — вывести информацию о группах;
- `-g`, `--groups=<группы>` — вывести пользователей, входящих в <группы>;
- `-L`, `--last `— показать информацию о последнем входе пользователя;
- `-n,` `--newline` — выводить каждый блок информации с новой строки;
- `-p`, `--pwd` — вывести информацию, связанную с входом по паролю;
- `-s`, `--system-accs` — вывести системных пользователей;
- `-u`, `--user-accs` — отобразить пользовательские аккаунты.

## Создание учётной записи пользователя

Основная утилита для создания учётной записи пользователя — `useradd`

Выполнение команды `useradd` без параметров создаст пользователя и первичную группу с тем же именем. В файле `/etc/passwd` у созданного пользователя будет указан домашний каталог с его именем, при этом в файловой системе такой каталог создан не будет. Чтобы его создать, вместе с `useradd` следует использовать параметр `-m`.

По умолчанию для новых пользователей вход в систему заблокирован. Об этом говорит восклицательный знак в строке с пользователем в файле `/etc/shadow`.

![[Pasted image 20260513174552.png]]

Для того чтобы пользователь получил возможность входа в систему, необходимо задать ему пароль с помощью утилиты `passwd`:

﻿ `sudo passwd user2`

После ввода пароля пользователь сможет выполнить вход в систему.

Применение утилиты `passwd` позволяет установить, изменить пароль пользователя, а также настроить атрибуты, связанные с жизненным циклом пароля. 

Основные параметры утилиты `useradd`:
- `-m` — Создать новому пользователю домашний каталог
- `-M` — Не создавать новому пользователю домашний каталог
- `-d` — Указать домашний каталог
- `-s` — Указать командный интерпретатор
- `-u` — Задать UID
- `-g` — Задать имя или ID основной (первичной) группы
- `-G` — Задать список дополнительных (вторичных) групп
- `-c` — Комментарий (GECOS)
- `-k` — Задать альтернативный каталог для файлов с настройками

Перечень доступных параметров утилиты можно отобразить, выполнив команду  `useradd -h` .

Примеры:

1. Создание обычной учётной записи пользователя:
﻿  `sudo useradd -s /bin/bash -d /home/user1 -m user1`  ﻿
2. Создание системной учётной записи пользователя:
﻿ `sudo useradd -u 666 -s /usr/sbin/nologin -d /etc/myserver -M myserver` ﻿

При создании учётной записи пользователя файлы настроек (.profile, .bashrc и др.) копируются из каталога `/etc/skel` в домашний каталог пользователя.

Другим способом добавления пользователя в систему является использование высокоуровневой утилиты `adduser`. В отличие от ранее рассмотренной команды `useradd`, которая создает в системе только самого пользователя и не учитывает какие-либо дополнительные настройки, в том числе создание домашнего каталога, команда `adduser` учитывает политики дистрибутива по желаемому `UID` и `GID`, создаёт домашний каталог, копирует туда файлы по умолчанию и запускает специальные сценарии работы пользователя. Настройки по умолчанию для утилиты `adduser` хранятся в файле `/etc/adduser.conf`. Данная утилита позволяет создать пользователя в несколько шагов, одним из которых является ввод пароля.

![[Pasted image 20260513174838.png]]

Команды `useradd` и `adduser` предназначены для добавления в систему только одной учётной записи одновременно. Чтобы иметь возможность добавить (или обновить) сразу несколько учётных записей пользователей в систему (в так называемом пакетном режиме), можно воспользоваться командой `newusers`. Информация по учётным записям при этом может считываться из стандартного потока ввода (по умолчанию) или файла.

Чтобы создать новых пользователей пакетом из файла, нужно предварительно создать такой файл, например `users.txt`, и добавить в него сведения об учётных записях пользователей в следующем формате:

`user_name:user_passwd:user_uid:user_gid:user_gecos:user_dir:user_shell`

где элементы строки обозначают:
- `user_name` — имя пользователя;
- `user_passwd` — пароль пользователя;
- `user_uid` — идентификатор пользователя;
- `user_gid` — идентификатор группы пользователя;
- `user_gecos` — определяет разделы комментариев;
- `user_dir` — определяет домашний каталог пользователя;
- `user_shell` — определяет пользовательскую оболочку по умолчанию.

Данный файл следует сохранить, после чего выполнить команду newusers с данным входным файлом, чтобы сразу добавить в систему вышеуказанные учётные записи пользователей:  `sudo newusers users.txt` .

Сначала команда newusers пытается создать или обновить учётные записи, а затем записать эти изменения в пользовательские или групповые базы данных. Чтобы убедиться, что учётные записи пользователей были добавлены, следует проверить файлы `/etc/passwd` и `/etc/groups`:  `cat /etc/passwd` .

## Изменение учётной записи пользователя

Изменение параметров существующей учётной записи пользователя выполняется с помощью утилиты `usermod`. Перечень доступных параметров утилиты можно отобразить, выполнив команду  `usermod -h` .

Основные параметры:
- `-c` — Новый комментарий GECOS
- `-d` — Установить новый домашний каталог
- `-g` — Новая первичная группа
- `-G` — Список вторичных групп
- `-a` — Добавить пользователя в дополнительные группы, указанные в параметре `-G`, не удаляя пользователя из других групп
- `-l` — Изменить имя пользователя
- `-L` — Заблокировать пользователя
- -`u` — Установить пользователю новый UID
- `-U` — Разблокировать пользователя

При использовании параметра `-G` без параметра `-a` пользователь будет добавлен в перечисленные группы, но удалён из всех остальных.

Пример добавления пользователя в группу:

﻿ `sudo usermod -aG astra-admin user1`

## Удаление учётной записи пользователя

Для удаления пользователя следует воспользоваться утилитой `userdel`. Использование утилиты с параметром `-r` позволяет удалить из системы домашний каталог удаляемого пользователя и почтовую информацию (почтовый ящик), хранящуюся в `/var/mail`.

Перед удалением учётной записи пользователя следует решить:
- что сделать с файлами пользователя (можно воспользоваться командой `find / -user …`, чтобы найти все файлы);
- требуется ли удалить основную (первичную) группу пользователя (обычно приватная основная группа удаляется автоматически).

>   Вместо удаления пользователя рекомендуется использовать его блокировку. Заблокировать пользователя можно:
>   - указав дату окончания действия учётной записи 2 января 1970 г., используя параметр `-e`:  `sudo usermod -e 1 user` ;
>   - используя параметр `-L`:  `sudo usermod -L user` .
