## Сети на основе стека TCP/IP

> Стек протоколов TCP/IP — это иерархически организованный набор сетевых протоколов, достаточный для организации взаимодействия узлов в сети.

Для описания технических спецификаций сетевых протоколов и сервисов используются стандарты **Request for Comments (RFC)**. Согласно **rfc1122**, стек интернет-протоколов состоит из четырёх уровней
- прикладной уровень (Application Level);
- транспортный уровень (Transport Level);
- межсетевой уровень (Internet Level);
- канальный уровень (Link Layer).

**Прикладной уровень** протоколов интернета — это уровень, через который пользовательские программы взаимодействуют с сетью. Протоколы прикладного уровня разделяются на две категории:
- пользовательские протоколы (`SSH`, `Telnet`, `FTP`, `SMTP`), которые предоставляют услуги непосредственно пользователям;
- протоколы поддержки (`DNS`, `BOOTP`, `SNMP`), которые обеспечивают общие системные функции.

Протоколы **транспортного уровня** используются для обеспечения взаимодействия сетевых приложений. На транспортном уровне определяются адреса сетевых приложений в рамках хоста — порты. В настоящее время существует два основных протокола транспортного уровня:
- **TCP** (Transmission Control Protocol) — предназначен для надёжной и гарантированной передачи (доставки) данных между хостами в компьютерных сетях с коммутацией пакетов и между сетями через промежуточные системы;
- **UDP** (User Datagram Protocol) — используется для передачи данных в режиме реального времени, таких как потоковое видео и аудио. Протокол не гарантирует надёжную доставку данных и порядок их поступления.

**Межсетевой уровень** отвечает за маршрутизацию пакетов между сетями, т. е. за адресацию `IP`, с помощью которой возможно идентифицировать и локализовать устройства в сети Интернет. Также он определяет, какой путь должны проходить пакеты от отправителя к получателю. Основной протокол межсетевого уровня — это Internet Protocol (`IP`).

**Канальный уровень** предназначен для обмена данными между соседними устройствами внутри одного сетевого сегмента (локальной сети). Для связи с непосредственно подключёнными к нему сетями хост должен поддерживать коммуникационный протокол, используемый для обмена данными с сетью.

## IP-адреса

Каждому сетевому интерфейсу должен быть присвоен **IP-адрес**. Его можно назначить статически, динамически, а также с помощью службы **DHCP**.

Существует две версии IP-протокола, которым соответствуют разные форматы адресов:
- `IPv4` (Internet Protocol version 4) — четыре байта. Записывается в виде четырёх десятичных чисел, разделённых точками (например, `192.168.1.1`), где каждое число может варьироваться от `0` до `255`.
- `IPv6` (Internet Protocol version 6) — шестнадцать байтов (`128` бит). Записывается в виде восьми чисел в шестнадцатеричном формате, разделяемых двоеточиями (например, `2001:0db8:85a3:0000:0000:8a2e:0370:7334`).

Принцип работы `DHCP`. Этот процесс называют **DORA**:
- `Discovery` (поиск) — клиент отправляет широковещательный запрос на поиск DHCP-сервера;
- `Offer` (предложение) — сервер предлагает клиенту IP-адрес и другие параметры;
- `Request` (запрос) — клиент подтверждает выбор предложенного адреса;
- `Acknowledgement` (подтверждение) — сервер подтверждает выделение адреса.

**Первый** адрес в сети — адрес самой сети. **Последний** адрес — адрес всех хостов в данной сети (широковещательный адрес).

Какой раздел IP-адреса относится к идентификатору сети, а какой — к идентификатору хоста, зависит от класса адреса или маски подсети, если используется подсетевое разделение.

**Маска сети** — это числовая комбинация, используемая для разделения IP-адреса на две части: **идентификатор сети** (`Network ID`) и **идентификатор хоста** (`Host ID`). Например, в домашних или малых офисных сетях обычно применяется маска `255.255.255.0`. Это означает, что первые три числа в IP-адресе определяют сеть, а последнее число — конкретный хост внутри этой сети.
