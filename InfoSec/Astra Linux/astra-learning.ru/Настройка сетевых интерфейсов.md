## Сети на основе стека TCP/IP

> Стек протоколов TCP/IP — это иерархически организованный набор сетевых протоколов, достаточный для организации взаимодействия узлов в сети.

Для описания технических спецификаций сетевых протоколов и сервисов используются стандарты **Request for Comments (RFC)**. Согласно **rfc1122**, стек интернет-протоколов состоит из четырёх уровней
- прикладной уровень (Application Level);
- транспортный уровень (Transport Level);
- межсетевой уровень (Internet Level);
- канальный уровень (Link Layer).

**Прикладной уровень** протоколов интернета — это уровень, через который пользовательские программы взаимодействуют с сетью. Протоколы прикладного уровня разделяются на две категории:
- пользовательские протоколы (`SSH`, `Telnet`, `FTP`, `SMTP`), которые предоставляют услуги непосредственно пользователям;
- протоколы поддержки (`DNS`, `BOOTP`, `SNMP`), которые обеспечивают общие системные функции.

Протоколы **транспортного уровня** используются для обеспечения взаимодействия сетевых приложений. На транспортном уровне определяются адреса сетевых приложений в рамках хоста — порты. В настоящее время существует два основных протокола транспортного уровня:
- **TCP** (Transmission Control Protocol) — предназначен для надёжной и гарантированной передачи (доставки) данных между хостами в компьютерных сетях с коммутацией пакетов и между сетями через промежуточные системы;
- **UDP** (User Datagram Protocol) — используется для передачи данных в режиме реального времени, таких как потоковое видео и аудио. Протокол не гарантирует надёжную доставку данных и порядок их поступления.

**Межсетевой уровень** отвечает за маршрутизацию пакетов между сетями, т. е. за адресацию `IP`, с помощью которой возможно идентифицировать и локализовать устройства в сети Интернет. Также он определяет, какой путь должны проходить пакеты от отправителя к получателю. Основной протокол межсетевого уровня — это Internet Protocol (`IP`).

**Канальный уровень** предназначен для обмена данными между соседними устройствами внутри одного сетевого сегмента (локальной сети). Для связи с непосредственно подключёнными к нему сетями хост должен поддерживать коммуникационный протокол, используемый для обмена данными с сетью.

## IP-адреса

Каждому сетевому интерфейсу должен быть присвоен **IP-адрес**. Его можно назначить статически, динамически, а также с помощью службы **DHCP**.

Существует две версии IP-протокола, которым соответствуют разные форматы адресов:
- `IPv4` (Internet Protocol version 4) — четыре байта. Записывается в виде четырёх десятичных чисел, разделённых точками (например, `192.168.1.1`), где каждое число может варьироваться от `0` до `255`.
- `IPv6` (Internet Protocol version 6) — шестнадцать байтов (`128` бит). Записывается в виде восьми чисел в шестнадцатеричном формате, разделяемых двоеточиями (например, `2001:0db8:85a3:0000:0000:8a2e:0370:7334`).

Принцип работы `DHCP`. Этот процесс называют **DORA**:
- `Discovery` (поиск) — клиент отправляет широковещательный запрос на поиск DHCP-сервера;
- `Offer` (предложение) — сервер предлагает клиенту IP-адрес и другие параметры;
- `Request` (запрос) — клиент подтверждает выбор предложенного адреса;
- `Acknowledgement` (подтверждение) — сервер подтверждает выделение адреса.

**Первый** адрес в сети — адрес самой сети. **Последний** адрес — адрес всех хостов в данной сети (широковещательный адрес).

Какой раздел IP-адреса относится к идентификатору сети, а какой — к идентификатору хоста, зависит от класса адреса или маски подсети, если используется подсетевое разделение.

**Маска сети** — это числовая комбинация, используемая для разделения IP-адреса на две части: **идентификатор сети** (`Network ID`) и **идентификатор хоста** (`Host ID`). Например, в домашних или малых офисных сетях обычно применяется маска `255.255.255.0`. Это означает, что первые три числа в IP-адресе определяют сеть, а последнее число — конкретный хост внутри этой сети.

Маска сети используется для подсетевого разделения, что позволяет разбивать крупные сети на более мелкие подсети. Это обеспечивает гибкое управление IP-адресами, повышает безопасность и облегчает маршрутизацию в сложных сетевых структурах.

## Протокол IPv4

`IPv4` — это четвёртая версия интернет-протокола, которая используется для передачи данных в сети Интернет. Он присваивает каждому устройству уникальный 32-битный IP-адрес, состоящий из четырёх групп чисел от `0` до `255`, разделённых точками (например, `192.168.1.1`). Хотя этот протокол всё ещё широко используется, его адресное пространство почти исчерпано.

В сетях `IPv4` поддерживаются три типа адресов:
- `Unicast` — единичный адрес, который соответствует единственному устройству в сети. Unicast-сообщения будут доставлены только этому устройству;
- `Multicast` — адреса для группы устройств. Multicast-сообщения будут доставлены всем устройствам в группе. Обычно используется для потоковых медиа и групповой связи;
- `Broadcast` — адрес указывает на все устройства в текущей IPv4-сети. Broadcast-сообщения будут доставлены всем устройствам, входящим в эту сеть. Адрес `255.255.255.255` в `IPv4` используется для ограниченного широковещания (limited broadcast), но он не предназначен для рассылки на устройства в разных IPv4-сетях, а только для всех узлов в пределах одной и той же подсети, в которой находится источник пакета.

Адрес IPv4 состоит из двух основных частей:
- **идентификатор сети** (Network ID) — часть адреса, указывающая на конкретную сеть в интернете. Все хосты в одной и той же сети имеют одинаковый идентификатор сети;
- **идентификатор хоста** (Host ID) — часть адреса, которая указывает на конкретный компьютер (или иную машину) в сети. Идентификатор хоста уникален в пределах своей сети.

## Протокол IPv6

`IPv6` — это новая версия интернет-протокола, призванная решить проблему исчерпания адресов `IPv4`. Его спецификация описана в `RFC 8200`.

Ключевые особенности и отличия от `IPv4`:
- Гораздо большее адресное пространство: длина адреса составляет `128` бит (по сравнению с `32` битами в IPv4). Это позволяет иметь `2^128` (около `3,4×10^38`) уникальных адресов.
	>Пример записи: `2001:0db8:85a3:0000:0000:8a2e:0370:7334`. Для сокращения допускается удаление ведущих нулей в каждом хекстете и замена одной последовательности нулевых хекстетов на двойное двоеточие (::): `2001:db8:85a3::8a2e:370:7334`.
- Упрощённый заголовок — заголовок `IPv6` фиксированного размера (`40` байт), что упрощает и ускоряет его обработку маршрутизаторами. Отсутствуют поля для фрагментации (она реализована иначе), контрольная сумма заголовка (расчёт возложен на вышележащие протоколы).
- Встроенная безопасность (`IPsec`) — поддержка `IPsec` (шифрование и аутентификация на сетевом уровне) является обязательной для реализации `IPv6`, хотя её использование остаётся опциональным.
- Эффективная маршрутизация — иерархическая структура адресации и агрегация маршрутов, заложенные в IPv6, позволяют сохранять таблицы маршрутизации компактными.    
- Автоконфигурация:
	- Stateless Address Autoconfiguration (`SLAAC`) — позволяет хосту автоматически сконфигурировать свой адрес на основе префикса, рассылаемого маршрутизатором, и своего MAC-адреса. Адрес маршрута по умолчанию выдаётся маршрутизатором;
	- Stateful `DHCPv6` — аналог `DHCP` для `IPv6`, когда адреса назначаются централизованно сервером.

Типы адресов IPv6:
- **Unicast** — адрес, идентифицирующий один интерфейс. Пакет доставляется только этому интерфейсу:
- **Global Unicast Address** (`GUA`) — публичные адреса, маршрутизируемые в интернете (аналогичны публичным IPv4). Диапазон — `2000::/3`.
- **Link-Local Address** (`LLA`) — адреса, используемые для связи только в пределах одного канального сегмента сети (аналогичны `169.254.0.0/16` в `IPv4`). Все интерфейсы имеют `LLA`, сгенерированный на основе `FE80::/10`.
- **Multicast** — адрес, идентифицирующий группу интерфейсов. Пакет доставляется всем интерфейсам группы. Широковещательных (Broadcast) адресов в IPv6 нет, их функцию полностью выполняет multicast.
- **Anycast** — адрес, назначенный нескольким интерфейсам (обычно на разных устройствах). Пакет доставляется ближайшему (с точки зрения метрики маршрутизации) интерфейсу с этим адресом.
