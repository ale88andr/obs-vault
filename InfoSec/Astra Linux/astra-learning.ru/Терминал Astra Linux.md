**Терминал** - интерфейс для последовательного ввода и вывода данных, а также для взаимодействия пользователя с операционной системой (ОС) через командную строку.

По умолчанию системной консолью является активный виртуальный терминал, на котором в данный момент работает пользователь.

Виртуальные терминалы используют аппаратные устройства — дисплей и клавиатуру. При этом эмулятор терминала реализован в ядре ОС. При стандартной установке Astra Linux с поддержкой графического окружения инициализируются семь виртуальных терминалов. На седьмом виртуальном терминале запускается графическая оболочка. В случае установки без графического окружения стартует шесть виртуальных терминалов. Переключение между виртуальными терминалами осуществляется с помощью комбинации клавиш `Ctrl + Alt + F1...F7`. Виртуальные терминалы именуются как `tty1`, `tty2` и т. д. Каждому виртуальному терминалу соответствует файл с тем же именем в каталоге `/dev`.

> Аббревиатура `TTY` расшифровывается как **TeleTYpe** — печатные машинки для передачи сообщений, которые использовались в качестве системной консоли на больших электронных вычислительных машинах первых поколений.

При работе в графической оболочке для доступа к системной консоли используются специальные программы — эмуляторы терминала. Они предоставляют доступ к псевдотерминалам.

> Количество виртуальных терминалов задается в конфигурационном файле `/etc/default/console-setup` — в параметре `ACTIVE_CONSOLES="/dev/tty[1-6]"`.

Процессы, работающие в фоновом режиме, должны блокироваться, как только они захотят вывести какой-либо текст на экран. Аналогичным образом набираемый пользователем текст должен передаваться только активной в данный момент программе. Операционная система реализует все эти функции при помощи драйвера `TTY`.

> **Процесс** — это экземпляр запущенной программы, представляющий собой активную работу программы в операционной системе. В него входят код программы, её текущее состояние (включая состояние процессора), данные, используемые программой, и выделенные ей системные ресурсы (память, открытые файлы и т. д.).

![[Pasted image 20260513101830.png]]

**Псевдотерминал** (`PTY`) представляет собой два виртуальных устройства, которые находятся в памяти ядра и могут передавать друг другу данные:

- **основной PTY** (Master PTY, PTM), подключённый к программному эмулятору терминала (находится в пользовательском пространстве);
- **подчинённый PTY** (Slave PTY, PTS), который взаимодействует с пользовательским процессом.

В качестве программных эмуляторов терминала могут выступать как графические приложения (`xterm`, `fly-term`), так и клиентские программы удалённого терминального доступа (`ssh`, `telnet`).

![[Pasted image 20260513102800.png]]

Для каждого псевдотерминала динамически генерируется номер и создаётся файл устройства с именем, соответствующим номеру псевдотерминала. Файл псевдотерминала обеспечивает доступ пользовательского процесса к драйверу `TTY` (подчинённому (slave) PTY, PTS) и размещается в каталоге `/dev/pts`. Например, `/dev/pts/3` — файл третьего псевдотерминала.

### Начало работы

В случае открытия программы Терминал работа по умолчанию будет выполняться от имени текущего пользователя.

После успешного входа отобразится информация о дате последнего входа, а также приглашение командной строки.

Для выхода достаточно ввести `exit` или `logout` и нажать `Enter`.

Независимо от вида терминала после авторизации по умолчанию запускается командный интерпретатор (`bash`). Это программа, которая анализирует пользовательский ввод и предоставляет интерфейс взаимодействия с системой.

Командный интерпретатор выводит приглашение, после которого пользователь может ввести команду.

Приглашение командной строки состоит:
- из имени текущего пользователя;
- имени текущего компьютера (хоста);
- текущего каталога. Домашний каталог текущего пользователя обозначается символом тильды (`~`);
- символа, обозначающего работу в качестве обычного пользователя (`$`) или суперпользователя (`#`).

![[Pasted image 20260513103918.png]]

Во время выполнения запущенный процесс может выводить информацию на терминал (дисплей). Процесс может завершиться без вывода информации на дисплей. Во многих случаях это означает, что процесс выполнился успешно.

### Структура команды

Структура командной строки имеет следующий вид:

﻿ `имя_команды параметры аргументы`

**Параметры** (options, опции) влияют на поведение команды.

**Аргументы** — объекты, на которые команда воздействует. Часто в качестве объектов выступают файлы и каталоги.

![[Pasted image 20260513104115.png]]

Разделителем между частями командной строки является пробельный символ — пробел или табуляция. Лишнее количество пробельных символов в командной строке игнорируется.

Примеры параметров:

﻿ `-f имя_файла` ﻿
﻿ `--file=имя_файла`

Интерпретатор `bash` умеет дополнять имена файлов, переменных, домашних каталогов пользователей.

>   Файлы, описывающие дополнение имён для разных команд, расположены в `/usr/share/bash-completion/completions`.

### История команд

Каждая выполненная команда сохраняется в истории командного интерпретатора. При его запуске история команд из файла (по умолчанию — `~/.bash_history`) загружается в буфер, при выходе из командного интерпретатора команды из буфера добавляются в файл.

Вывод всей истории команд:

﻿ `history` ﻿

Вывод указанного количества последних строк:

﻿ `history число` ﻿

Добавить в файл историю команд из буфера:

﻿ `history -a` ﻿

Нажатие клавиши вверх отобразит последнюю выполненную команду. Навигация по списку выполняется клавишами вверх и вниз.

Выполнение команды из истории по номеру:

﻿ `!номер` ﻿

Выполнение предыдущей команды:

﻿ `!!` ﻿

Также к двойному восклицательному знаку можно дописывать команду справа или слева. Например, если требуется выполнить ту же команду, но с правами суперпользователя:

﻿ `sudo !!`

### Маскировка специальных символов

Ряд символов воспринимаются интерпретатором командной строки не буквально, а как указания произвести некоторые операции. К наиболее часто используемым спецсимволам можно отнести:

- `$` — при использовании переменных;
- `>`, `<` и `|` — при работе с потоками;
- `;` — при последовательном выполнении команд и прочие.

### Переменные

**Переменная** — именованная область памяти, значение которой может изменяться.

Создание переменной с присвоением ей значения имеет следующий синтаксис:

﻿  `VAR_NAME=значение`

Для получения значения переменной используется специальный символ $:

﻿  `echo $VAR_NAME`  ﻿

или

﻿  `echo ${VAR_NAME}`  ﻿

Удаление переменной осуществляется командой:

﻿  `unset VAR_NAME`

Использование фигурных скобок при получении значения переменной требуется, если нужно явно выделить имя переменной и избежать неправильной трактовки строки после символа $ командным интерпретатором.

Пример: в первом случае будет выведено значение переменной A1, во втором — значение переменной A, к которому будет добавлен символ 1.

﻿ `echo $A1` ﻿

﻿ `echo ${A}1`

![[Pasted image 20260513105832.png]]

Переменные можно разделить на два класса:

- переменные командного интерпретатора bash (shell variables);
- переменные окружения (environment variables).

Интерпретатор `bash` поддерживает набор предопределённых переменных (см. `man bash`, секция Shell Variables, переменные командного интерпретатора). Наиболее популярными являются:

- `$LOGNAME, $USER` — имя пользователя;
- `$UID` — идентификатор пользователя;
- `$PATH` — перечень каталогов для поиска команд;
- `$HOSTNAME` — имя хоста;
- `$HOME` — домашний каталог пользователя;
- `$$` — идентификатор текущего процесса;
- `$PPID` — идентификатор родительского процесса;
- `$PWD` — текущий каталог;
- `$LANG` — локаль.

Переменные окружения передаются в процессы, порождённые командным интерпретатором, а переменные командного интерпретатора — нет.

### Многострочные команды, последовательное и условное выполнение

В одной строке можно написать несколько команд, разделяя их символом ;. При этом команды будут выполняться последовательно:

﻿ `команда1; команда2; команда3`

Также команды могут быть заключены в круглые скобки. Это позволит получить результат выполнения сразу всех команд. Если требуется, например, записать результат работы нескольких команд в один файл, то команды заключаются в круглые скобки, после которых ставится закрывающая угловая скобка и указывается имя файла:

﻿ `(команда1; команда2) > файл`

Условное выполнение команд (выполнение команды зависит от успешности выполнения предыдущей команды) осуществляется путем вставки символов && (логическое И) и || (логическое ИЛИ) между командами.

﻿ `команда1 || команда2 && команда3` ﻿

Использование логического ИЛИ между командами означает, что если первая команда:

- была выполнена успешно, то вторая команда не выполняется;
    
- завершилась неудачей, то выполняется вторая команда.
    

Пример: если не смогли перейти в каталог D1, то создаём его:

﻿ `cd D1 || mkdir D1` ﻿

Использование логического И между командами означает, что вторая команда выполняется только тогда, когда первая команда выполнена успешно.

Пример: если удалось создать каталог, то переходим в него:

﻿ `mkdir D1 && cd D1` ﻿

Порядок выполнения условий можно контролировать, используя круглые скобки.

Каждая выполненная команда передаёт в командный интерпретатор код завершения (exit status). Код завершения, равный 0, — успех, положительное значение кода завершения — неудача. Коды завершения для разных команд можно найти при помощи справочной команды man. Чтобы узнать код завершения предыдущей команды, следует выполнить:

﻿ `echo $?`
