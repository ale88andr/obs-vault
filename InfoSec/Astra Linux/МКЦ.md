Мандатный контроль целостности - это про защиту ОС и основных её служб

![[Pasted image 20260204132653.png]]

**Субъект доступа (процесс)** — активный компонент ОС права доступа
которого регламентируются политиками управления доступом.

**Объект доступа (файл)** — пассивный компонент ОС, доступ к которому
регламентируется политиками управления доступом. Может быть составным
объектом (контейнером или каталогом, содержащим другие объекты
и контейнеры или каталоги, к которым по отдельности можно применять
политики управления доступом).

Уровни целостности:
- Высокий - `63`
- Низкий - `0`

> Непривилегированным пользователям по умолчанию присваивается нулевой уровень целостности, администратору — максимальный уровень целостности 63. Системными службами зарезервированы четыре изолированных уровня целостности.

МКЦ влияет на запись (смена владельца, смена прав, создание или изменение файлов)

Если на объекте уровень целостности - `0` (`Низкий`) - то, его может менять субъект работающий под уровнем `0` (`Низкий`) или `63` (`Высокий`)

Если на объекте уровень целостности - `63` (`Высокий`) - то, его может менять субъект работающий только под уровнем `63` (`Высокий`)

> Субъект с определенным уровнем целостности может получить доступ на запись к сущности, если его уровень целостности не ниже уровня целостности сущности.

## Смена уровня целостности

Через GUI: `Параметры системы` -> `Пользователи`

![[Pasted image 20260130161027.png]]

Через терминал:

1. просмотр меток:

```shell
sudo pdpl-user <имя_пользователя>
```

![[Pasted image 20260130161309.png]]

При этом показывается минимальный и максимальный уровень целостности при котором может работать пользователь

2. изменение метки:

```shell
sudo pdpl-user -i 63 <имя_пользователя>
```
