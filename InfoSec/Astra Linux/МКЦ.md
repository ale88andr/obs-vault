Мандатный контроль целостности - это про защиту ОС и основных её служб

Уровни целостности:
- Высокий - `63`
- Низкий - `0`

МКЦ влияет на запись (смена владельца, смена прав, создание или изменение файлов)

Если на объекте уровень целостности - `0` (`Низкий`) - то, его может менять субъект работающий под уровнем `0` (`Низкий`) или `63` (`Высокий`)

Если на объекте уровень целостности - `63` (`Высокий`) - то, его может менять субъект работающий только под уровнем `63` (`Высокий`)

## Смена уровня целостности

Через GUI: `Параметры системы` -> `Пользователи`

![[Pasted image 20260130161027.png]]

Через терминал:

1. просмотр меток:

```shell
sudo pdpl-user <имя_пользователя>
```

![[Pasted image 20260130161309.png]]

При этом показывается минимальный и максимальный уровень целостности при котором может работать пользователь

2. изменение метки:

```shell
sudo pdpl-user -i 63 <имя_пользователя>
```
